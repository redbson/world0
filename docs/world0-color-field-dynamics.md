# World 0 色场动力学设计

## 1. 设计定位

本文档定义一种面向 World 0 的**概念域色场机制**。

它不是传统意义上的“标签系统”：

- 不是先把概念硬分到某个域
- 不是给概念直接贴一个固定颜色
- 不是依靠大量手工阈值和经验参数维持行为

它的目标是：

1. 从概念关系网络中自然形成局部群落
2. 让稳定群落自发地产生颜色
3. 让颜色像液体中的染料一样沿关系扩散
4. 让失去结构支撑的颜色随时间自然褪去
5. 让桥接概念保留混色，而不是被视为污染

这套机制应被理解为一种**动力系统**，而不是一套分类规则。

---

## 2. 核心思想

World 0 当前的基本对象是：

- `concept`
- `relation`
- `activation`
- `projection`

色场机制不是替代这些对象，而是叠加在它们之上的一个**结构涌现层**。

它的核心思想是：

**关系形成群落，群落生成颜色，颜色沿图扩散，失去支撑的颜色随时间褪去。**

因此：

- 颜色先属于群落，而不是先属于概念
- 概念只是承载颜色强度的节点
- 色彩传播不是标签复制，而是局部场的传播

---

## 3. 基本状态变量

建议把系统建模为以下几个基本量。

### 3.1 激活量 `activation`

表示概念当前的活跃程度。

它来自：

- 当前任务
- 历史强化
- 邻域传播

激活量不直接等于颜色，但会影响群落形成和颜色维持。

### 3.2 耦合量 `coupling`

表示概念之间的联结强度。

它不只是 relation weight，而是更广义的局部联结量，可以由以下因素自然组成：

- typed relation 的权重
- 共同激活历史
- 共现频率
- 时间上的邻近性

### 3.3 凝聚场 `cohesion field`

表示某个局部区域内部的结构凝聚程度。

它描述的是：

- 内部连接是否足够紧密
- 内外边界是否足够清晰
- 结构是否足够稳定

凝聚场用于判断一个区域是否正在形成群落。

### 3.4 色场 `color field`

表示某个群落颜色在概念图中的局部分布。

它是随时间变化的强度分布，不是单个离散标签。

对每个概念来说，它承载的是：

- 对一个或多个群落颜色的响应强度
- 随时间变化的成色、混色和褪色状态

## 4. 五阶段动力过程

### 4.1 无色阶段

系统初始时没有域颜色。

此时只积累：

- 概念
- 关系
- 激活
- 强化

也就是说，系统先长出结构，再长出色彩。

### 4.2 群落形成阶段

当若干概念之间的耦合长期高于周围背景时，局部群落开始出现。

群落判定不建议基于单一阈值，而建议基于以下自然量：

- 内部平均耦合强度
- 内部耦合 / 外部耦合 比
- 连续多个周期的结构稳定性
- 群落最小尺度

换句话说，群落不是一次观察得到的，而是局部结构持续稳定后的涌现结果。

### 4.3 颜色诞生阶段

只有当一个群落持续稳定后，它才获得自己的颜色身份。

颜色不是预定义域名，而是群落的结构身份。

后续可以由系统或人类把这个颜色解释为：

- backend-like
- ml-like
- devops-like

但命名发生在颜色生成之后，而不是之前。

### 4.4 颜色扩散阶段

一旦群落获得颜色，颜色就开始沿关系传播。

传播原则：

- 群落内部扩散快
- 穿越群落边界扩散慢
- 耦合强的边扩散强
- 泛化关系扩散弱
- 跨边界时自然耗散

颜色传播是一个图上的扩散过程，不是概念标签复制。

### 4.5 颜色褪色阶段

如果一个概念逐渐脱离原有群落，它对该颜色的响应不会立刻消失，而会随时间自然退相。

褪色条件来自：

- 失去内部耦合支撑
- 连续多个周期不再接受该颜色输入
- 新群落颜色逐步占优

褪色是动力系统的组成部分，而不是垃圾清理器。

---

## 5. 群落生成机制

### 5.1 群落的基本定义

一个候选群落应满足：

1. 有最小节点规模
2. 内部耦合显著高于外部
3. 结构在时间上持续存在
4. 至少部分节点在当前阶段保持活跃

这意味着：

- 短暂共现不形成群落
- 临时任务簇不形成群落
- 噪声概念不会轻易变成染色源

### 5.2 群落的层级结构

每个群落内部可区分为：

- `core`
- `inner ring`
- `boundary`

颜色在群落内部的生成应遵循：

- 从核心开始最纯
- 向内圈逐步增强
- 边界层颜色较弱且更容易波动

---

## 6. 成色机制

概念不会因为“属于某群落”而立刻获得满颜色。

成色是一个逐渐沉积的过程。

建议把每个概念对群落颜色的状态理解为一种**颜色密度**或**沉积强度**。

成色来源包括：

- 群落核心直接输入
- 群落内部邻域扩散
- 与群落的持续结构耦合

成色的自然特征：

- 群落核心成色快
- 边界节点成色慢
- 临时接触不会留下很深颜色

---

## 7. 扩散机制

### 7.1 扩散原则

颜色扩散应满足：

- 局部传播
- 有损耗
- 随耦合增强而变强
- 随边界穿越而变弱

扩散过程可以被理解为：

`颜色变化 = 染入 + 邻域扩散 - 外流 - 耗散`

### 7.2 扩散的自然驱动因素

不建议直接手写固定扩散率，而应使用局部结构归一化量：

- 边耦合相对于邻域平均耦合的比例
- 节点当前颜色浓度相对于邻居的梯度
- 局部凝聚度
- 是否处于群落边界

因此扩散速度是结构自然导出的，而不是手工调一个统一常数。

### 7.3 关系类型的角色

关系类型不应只影响激活，也应影响色场传播。

但传播权重不应简单复用 activation 权重。

因为：

- 激活表示“任务相关性”
- 染色表示“结构归属传播”

这两者相关，但不是同一件事。

---

## 8. 混色与桥接概念

桥接概念不是污染，而是边界层。

它们的典型特征是：

- 同时受到两个稳定群落的颜色输入
- 两侧结构支撑都真实存在
- 颜色强度接近或长期共存

桥接概念应被视为一种稳定态，而不是异常态。

建议为其定义独立状态：

- `bridge`
- `mixed-core`
- `boundary-mixed`

桥接概念的规则：

- 可以保留双主色
- 应弱于单域核心向外传播
- 不应成为全局污染泵

---

## 9. 褪色机制

### 9.1 褪色的触发

一个概念对某颜色开始褪色，不是因为“本轮没看到这个域”，而是因为它对该群落的结构支撑持续减弱。

可分为三阶段：

1. `弱脱离`
   颜色不再继续增强
2. `明确脱离`
   进入褪色态
3. `完全脱离`
   颜色分量从显式状态中移除

### 9.2 褪色的自然形式

褪色应采用弛豫或半衰期式衰减，而不是线性减去一个固定量。

原因：

- 更符合系统整体时间动力学
- 新染色退得快
- 深层颜色退得慢
- 不会造成硬截断

### 9.3 深色与浅色

同样的颜色，不同沉积深度应有不同褪色速度。

- 核心成员的深色退得慢
- 边界沾染的浅色退得快
- 临时接触留下的颜色最容易消失

### 9.4 按颜色分量独立褪色

对于混色节点，褪色必须是按颜色分量独立进行的。

例如：

- backend 色保留
- ml 色减弱

这样桥接节点才能自然地从双域态回到单域态。

---

## 10. 轻参数原则

这个系统不应依赖大量手工参数。

建议原则是：

- 参数只作为自然尺度的归一化器
- 不作为主行为驱动器

可接受的轻参数包括：

- 自然时间单位
- 最小群落规模
- 最小可见颜色分量
- 可视化时的显示压缩量

这些参数应尽量由系统当前统计态自然导出，例如：

- 最近一段时间的中位激活间隔
- 局部耦合的分位数
- 局部凝聚度背景均值

原则上：

**系统行为应由动力学决定，参数只负责把动力学映射到有限计算尺度。**

---

## 11. 防污染原则

在动力学框架下，防污染不应主要依赖硬阈值，而应依赖以下机制：

### 11.1 结构优先

只有稳定群落才能生色。

临时任务、零散概念、偶然共现不应直接产生颜色源。

### 11.2 守恒与耗散

颜色不是无限叠加，而应是一种有限场。

有输入时增强，无输入时褪去。

### 11.3 边界阻尼

穿越群落边界时，扩散自然减弱。

### 11.4 桥接不等于污染

混色节点若有真实双侧结构支撑，应视为边界层而非噪声。

### 11.5 弱颜色自动蒸发

极弱、长期无支撑的颜色分量应自然消失，而不是永久残留。

## 12. 可观测量

为了让这套系统可调试、可验证，建议长期暴露以下观测指标：

### 12.1 节点层

- 节点主色
- 节点混色度
- 节点颜色纯度
- 节点颜色变化速度
- 是否为桥接节点

### 12.2 群落层

- 群落规模
- 内部凝聚度
- 边界清晰度
- 群落寿命
- 颜色稳定度

### 12.3 全局层

- 平均每节点颜色分量数
- 平均颜色纯度
- 桥接节点比例
- 群落数量随时间变化
- 颜色污染率

这些观测量不是附属品，而是动力系统的必要诊断面。

## 13. 对 World 0 的价值

这套机制对 World 0 的价值不在于“让图更好看”，而在于提供一种新的结构语义层。

它可以帮助系统识别：

- 哪些概念已经形成稳定域团块
- 哪些概念是跨域桥
- 哪些概念正在脱离旧域
- 当前投影更靠近哪个局部概念世界

它还可以为未来提供三个方向：

1. 基于色场的 projection reranking
2. 基于桥接色的跨域推理
3. 基于褪色轨迹的概念迁移分析

---

## 14. 与当前简单染色方案的关系

如果仓库中已经存在“基于 task/domain 直接注色”的简单版本，应将其视为：

- 可视化实验层
- 初步验证层
- 过渡机制

而不是最终理论方案。

最终方案应从“外部注色”逐步过渡到“群落生色”。

建议过渡路径：

1. 保留简单注色，仅用于观察
2. 加入群落检测，但不接管渲染
3. 让群落颜色逐步替代 task/domain 颜色
4. 最终以群落色场为主，task 只作为 activation 上下文

## 15. 一句话定义

**World 0 的色场机制应被设计为一种由概念耦合、局部群落、颜色扩散与时间褪色共同驱动的自组织动力系统，而不是一个依赖外部标签和手工参数的分类系统。**

## 16. 数学附录：统一动力学形式

本节给出一个统一的数学表述，用于说明这套机制为什么是：

- 群落自发形成
- 颜色自发诞生
- 颜色沿图扩散
- 颜色在失去支撑后自然褪去
- 系统整体保持有界而不发生无限污染

这不是实现唯一形式，而是一套足够一般的理论框架。

### 16.1 图与状态

令概念系统在时间 `t` 上对应一个加权图：

`G_t = (V, K_t)`

其中：

- `V = {1, ..., n}` 是概念节点集合
- `K_t = (K_ij(t))` 是非负耦合矩阵
- `K_ij(t) >= 0`
- `K_ii(t) = 0`

`K_ij(t)` 不是单一 relation weight，而是综合联结量，可理解为：

- 显式关系权重
- 共现历史
- 共同激活历史
- 时间邻近性修正

定义节点度：

`d_i(t) = sum_j K_ij(t)`

定义随机游走矩阵：

`P_t = D_t^(-1) K_t`

其中 `D_t = diag(d_1(t), ..., d_n(t))`。

定义归一化图拉普拉斯：

`L_t = I - P_t`

它描述颜色和激活在图上的扩散几何。

### 16.2 关键状态变量

对每个概念 `i`，定义：

- `a_i(t)`：激活量
- `k_i(t)`：局部凝聚响应
- `c_i^alpha(t)`：颜色 `alpha` 在节点 `i` 上的强度

其中颜色 `alpha` 不预先对应某个手工域名，而是对应一个稳定群落产生的颜色模式。

## 17. 结构演化方程

### 17.1 耦合演化

令 `r_ij(t)` 表示显式关系输入，`a_i(t)a_j(t)` 表示共同活动带来的耦合增强，则可写：

`dK_ij/dt = Phi_ij(a, r, t) - K_ij / tau_K(t)`

其中：

`Phi_ij(a, r, t) = (r_ij(t) + a_i(t)a_j(t)) / (Med_{p,q}(r_pq(t) + a_p(t)a_q(t)) + eps)`

解释：

- 第一项表示关系和共同活动带来的联结增长
- 第二项表示无支撑时的自然弛豫
- `tau_K(t)` 不是手工固定常数，而应由系统当前统计态自然导出，例如最近强化间隔的中位数

这保证了“结构先形成，再生色”。

### 17.2 局部凝聚场

对每个节点集合 `S subset V`，定义导通率：

`h_t(S) = cut_t(S, V\\S) / min(vol_t(S), vol_t(V\\S))`

其中：

- `cut_t(S, V\\S) = sum_{i in S, j notin S} K_ij(t)`
- `vol_t(S) = sum_{i in S} d_i(t)`

若某局部子图长期满足：

- `h_t(S)` 足够低
- 内部耦合显著高于外部
- 结构持续时间达到 `O(tau_K)`

则称 `S` 为一个亚稳群落。

这意味着群落不是一次输入决定的，而是由结构长期自组织形成的。

## 18. 群落诞生与谱论解释

### 18.1 谱隙条件

令 `L_t` 的特征值按升序排列：

`0 = lambda_1 <= lambda_2 <= ... <= lambda_n`

若存在某个 `m` 使得：

`lambda_m << lambda_{m+1}`

则前 `m` 个慢模态对应 `m` 个近似不变的群落结构。

这给出了一个关键结论：

**颜色数量不应由外部手工指定，而应由图结构的慢模态数量自然决定。**

### 18.2 群落颜色源

一旦群落 `C_alpha` 稳定形成，就定义其颜色源项：

`s^alpha(t) in R^n_{>=0}`

满足：

- `s_i^alpha(t) > 0` 当且仅当 `i` 属于该群落核心区
- 若群落不稳定，则 `s^alpha(t) = 0`

因此：

- 没有稳定群落，就没有颜色源
- 短暂噪声不会成为一级染色源

## 19. 色场扩散与褪色方程

### 19.1 统一色场方程

对每个颜色 `alpha`，定义颜色场向量：

`c^alpha(t) = (c_1^alpha(t), ..., c_n^alpha(t))^T`

其演化满足：

`dc^alpha/dt = -eta^alpha(t) L_t c^alpha + s^alpha(t) - mu^alpha(t) odot c^alpha`

其中：

- `-eta^alpha(t) L_t c^alpha`：沿图扩散
- `s^alpha(t)`：群落核心持续染色
- `mu^alpha(t) odot c^alpha`：失去支撑后的褪色
- `eta^alpha(t)`：自然扩散尺度
- `mu^alpha(t)`：局部褪色率

这里 `odot` 表示逐分量乘法。

### 19.2 扩散尺度的自然化

不建议把 `eta^alpha(t)` 写成固定常数，而应让它由局部结构导出，例如：

`eta_i^alpha(t) ~ local_coupling_i / median_local_coupling`

含义是：

- 局部耦合越强，颜色扩散越快
- 穿越边界时，扩散自然变慢

### 19.3 褪色率的自然化

定义颜色 `alpha` 在节点 `i` 上的局部支撑量：

`q_i^alpha(t) = (P_t c^alpha(t))_i`

即邻域对该颜色的供给。

则褪色率可定义为：

`mu_i^alpha(t) = (1 / tau_c(t)) * (1 - q_i^alpha(t) / (Med_beta q_i^beta(t) + eps))_+`

其中：

- `(x)_+ = max(x, 0)`
- `tau_c(t)` 是自然颜色时间尺度，例如最近颜色更新间隔的中位数

解释：

- 若该颜色仍被邻域持续支撑，则 `mu_i^alpha` 小
- 若该颜色已失去支撑，则 `mu_i^alpha` 增大

这正对应“脱离某个域后会慢慢褪色”的设想。

## 20. 桥接概念的混色论证

设节点 `b` 同时连接两个稳定群落 `A` 和 `B`。

在准稳态下，颜色满足：

`eta^alpha L_t c^alpha + mu^alpha odot c^alpha = s^alpha`

对单个桥接节点做局部近似，可得：

`c_b^alpha ~= (sum_j P_bj c_j^alpha + s_b^alpha) / (1 + mu_b^alpha)`

因此，若节点 `b` 对两个群落的有效导通都非零，且量级接近，则：

- `c_b^A > 0`
- `c_b^B > 0`
- 当两侧输入近似平衡时，`c_b^A ~= c_b^B`

所以桥接概念自然形成混色态。

结论：

**混色节点不是异常，而是扩散方程在群落边界上的自然解。**

---

## 21. 有界性与非爆炸性

### 命题 1：颜色场保持非负

若：

- 初值 `c^alpha(0) >= 0`
- 源项 `s^alpha(t) >= 0`
- 扩散项由图拉普拉斯给出
- 褪色项 `mu_i^alpha(t) >= 0`

则对任意 `t >= 0`，都有：

`c^alpha(t) >= 0`

证明思路：

- 图拉普拉斯扩散不创造负浓度
- 源项非负
- 耗散项只减少已有正量
- 因而正正不变性成立

### 命题 2：颜色场有界

若源项有界，即存在常数 `M_alpha` 使得：

`||s^alpha(t)|| <= M_alpha`

则颜色场满足估计：

`||c^alpha(t)|| <= exp(-m_alpha t)||c^alpha(0)|| + integral_0^t exp(-m_alpha (t-u)) ||s^alpha(u)|| du`

其中 `m_alpha = inf_{i,t} mu_i^alpha(t) >= 0`。

因此：

- 若源项有限，则颜色场不会无限增长
- 若某颜色源消失，则该颜色最终衰减

这说明系统不会因为一次误染而永久无限污染。

---

## 22. 褪色与去污染论证

### 命题 3：无源时颜色指数衰减

若在某一阶段开始后，对颜色 `alpha` 有：

- `s^alpha(t) = 0`
- 邻域支撑持续减弱
- `mu_i^alpha(t) >= mu_0 > 0`

则有：

`||c^alpha(t)|| <= exp(-mu_0 (t-t_0)) ||c^alpha(t_0)||`

所以一旦某概念真正脱离某个群落，该颜色会自然褪去，而不是永久残留。

### 命题 4：短暂噪声源只留下短暂残迹

若某噪声颜色源仅在有限时间窗口 `[t_1, t_2]` 非零，则其对任意节点的影响满足：

`c_noise(t) <= integral_{t_1}^{t_2} G(t,u) s_noise(u) du`

其中 `G(t,u)` 是扩散-耗散系统的格林算子。

由于：

- 时间窗口有限
- 扩散有损耗
- 褪色项非零

所以噪声颜色的长期残留是有界且可衰减的。

这提供了防污染的数学基础。

## 23. 轻参数的数学含义

“轻参数”不意味着完全没有参数，而意味着：

- 参数不是主行为来源
- 参数由自然尺度归一化产生

建议只保留三类尺度：

### 23.1 结构时间尺度 `tau_K(t)`

用于描述耦合增长与回落的自然时间。

可由最近强化事件间隔的中位数给出。

### 23.2 颜色时间尺度 `tau_c(t)`

用于描述颜色形成与褪色的自然时间。

可由最近群落支撑更新的中位间隔给出。

### 23.3 可见性尺度

用于可视化或投影时压缩极小颜色分量。

它不是动力学参数，而是观测阈值。

因此，这个系统的参数应像“单位制”一样存在，而不是像“经验调节钮”一样主导行为。

## 24. 理论上的整体结论

把以上几部分合起来，可以得到一个统一结论：

1. 概念图通过局部耦合演化出亚稳结构
2. 亚稳结构通过低导通率和谱隙表现为群落
3. 稳定群落产生颜色源
4. 颜色在图上按反应-扩散-耗散方程传播
5. 桥接概念自然成为混色边界层
6. 失去支撑的颜色按时间弛豫并褪去
7. 因为系统满足正性、有界性和无源衰减性，所以不会发生无限污染

换句话说：

**World 0 的色场机制可以被严格理解为一个定义在慢变概念图上的群落驱动反应-扩散-弛豫系统。**

## 25. 对实现的约束意义

这个数学附录对工程实现提出了几个明确约束：

1. 不要先定义固定域名再给节点涂色
2. 不要把颜色仅实现成离散标签
3. 不要让扩散没有耗散项
4. 不要让褪色只靠硬删除
5. 不要让桥接概念被误当成污染
6. 不要让参数变成主行为来源

如果实现偏离这些约束，就会退化为：

- 分类器
- 打标签系统
- 依赖经验阈值的视觉效果层

而不再是一个真正的动力系统。

## 26. 生命体式成长路径

这套系统不应从一开始就接管 World 0 主逻辑。

更合理的定位是：

- 它是一个逐步发育的二级认知层
- 它需要足够长的结构积累过程
- 它必须先证明“自己稳定存在”，再证明“自己有认知价值”

因此建议采用生命体式成长路径。

### 26.1 萌芽期：只记录，不干预

能力边界：

- 记录耦合变化
- 识别候选群落
- 追踪潜在颜色源
- 记录褪色趋势

限制：

- 不影响 `project()`
- 不影响激活排序
- 不影响 agent 输出

目标：

- 验证局部结构是否真的会自然聚团

### 26.2 感知期：只可视化，不决策

能力边界：

- 展示群落结构
- 展示色场分布
- 展示桥接节点
- 展示褪色轨迹

限制：

- 仍不介入核心推理路径

目标：

- 验证色场是否可解释、可观测、可诊断

### 26.3 稳定期：只做解释，不做控制

能力边界：

- 解释某次 projection 更靠近哪个局部群落
- 识别桥接概念和迁移概念
- 为概念演化提供结构说明

限制：

- 只做解释层，不做排序层

目标：

- 验证它是否提供了真实的结构信息，而不是视觉错觉

### 26.4 共生期：弱影响主逻辑

能力边界：

- 对 projection 做轻微 rerank
- 对 bridge 概念做结构加权
- 在明确 task 语境下引入弱域一致性偏置

限制：

- 只能作为辅助信号
- 不能替代现有 activation / projection 机制

目标：

- 验证它是否能在可控范围内提升输出质量

### 26.5 成熟期：正式并行认知层

能力边界：

- 成为 projection 的正式二级结构信号
- 支持跨域桥接推理
- 支持概念迁移分析
- 支持长期局部世界演化诊断

前提：

- 前面几个阶段都已经被验证为稳定且有效

## 27. 有效性验证框架

这套系统不能只凭理论优雅就进入主路径。

必须有明确的验证框架，回答两个问题：

1. 它是否健康生长
2. 它是否提升 agent 的实际表现

### 27.1 早期验证：结构健康

在前 3 个阶段，主要验证的是“生命体是否健康”，而不是“是否已经提升 agent”。

核心问题：

- 是否形成稳定群落
- 颜色是否稳定出现
- 桥接节点是否合理
- 褪色是否自然发生
- 是否出现大面积污染

建议指标：

- 群落寿命
- 群落数量稳定度
- 节点平均纯度
- 混色节点比例
- 弱颜色蒸发率
- 桥接节点跨域真实性

### 27.2 后期验证：任务有效性

在共生期和成熟期，才评估它是否提升 agent。

推荐三组对照：

1. `No Memory`
   不使用 World 0，不给结构化上下文
2. `Raw History`
   给历史文本或摘要，但不用色场结构
3. `World0 + Color Field`
   给现有 projection，并注入色场解释或轻 rerank

### 27.3 推荐任务类型

最适合评估色场机制的不是单轮问答，而是：

- 多轮累积任务
- 跨域桥接任务
- 上下文切换任务
- 长距离召回任务
- 关系推理任务

### 27.4 推荐指标

可以分为四类：

#### 结构指标

- 群落凝聚度
- 群落边界清晰度
- bridge 稳定性
- 漂移概念比例

#### 投影指标

- projection relevance
- 域聚焦度
- 跨域桥接召回率
- 域污染率

#### agent 指标

- 任务成功率
- 关系识别率
- 长距离概念召回率
- token per success

#### 诊断指标

- 平均每节点颜色分量数
- 节点颜色熵
- 色场稳定时间
- 弱颜色残留率

### 27.5 准入原则

只有当它同时满足以下条件，才允许从“解释层”进入“弱控制层”：

1. 群落稳定性可重复出现
2. 桥接节点具有可解释性
3. 色场污染率低于可接受范围
4. 在 benchmark 中优于 `Raw History` 基线

也就是说，它必须证明自己优于“普通摘要”，而不是只优于“完全没有记忆”。

## 28. 优雅性与有效性判断

### 28.1 为什么它是优雅的

这套系统优雅的根本原因在于，它与 World 0 的哲学是一致的：

- relation-first
- context-sensitive
- projection-oriented
- structure before label

它把“域”从外部标签变成内部涌现，这比直接给节点打 `domain` 更符合 concept-world 的目标。

### 28.2 为什么它不一定立刻有效

但优雅不等于立刻有效。

它的工程有效性取决于：

- 图结构是否足够好
- 关系质量是否稳定
- 观测时间是否足够长
- 群落是否真的形成
- 最终是否改善 agent 输出

如果这些前提不成立，它可能会：

- 看起来很美
- 解释起来很强
- 但对实际任务增益不明显

因此应把它视为：

- 研究价值高
- 理论优雅度高
- 产品有效性需要逐步证明

### 28.3 工程判断原则

这套机制只有在下列条件下才算工程上也优雅：

- 状态变量少而统一
- 行为来自同一套动力学
- 参数来自自然尺度
- 诊断面完备
- 与现有系统并行生长，而不是推倒重来

否则，它会退化成：

- 规则堆
- 阈值堆
- 解释成本高但收益不明的复杂系统

## 29. 工程实现路线图

基于以上讨论，建议的实现路线图如下。

### 阶段 A：观测层

只增加：

- 耦合追踪
- 候选群落检测
- 色场状态缓存

不接入主 projection。

### 阶段 B：可视化层

增加：

- 群落视图
- 色场分布图
- 桥接概念高亮
- 褪色轨迹图

目标是看系统是否“长得对”。

### 阶段 C：解释层

为 projection 提供：

- 当前群落解释
- bridge 解释
- 域迁移解释

仍然不改排序。

### 阶段 D：弱影响层

允许：

- 小权重 rerank
- 轻量 bridge 加权
- 局部群落一致性偏置

必须以 benchmark 结果为准入依据。

### 阶段 E：正式并行层

在长期验证通过后，才允许成为 World 0 的正式二级结构层。
