# World 0：一种面向 LLM Agent 的概念世界认知层

## 摘要

大语言模型驱动的 Agent 在复杂任务中常常表现出两类结构性缺陷：其一，任务上下文主要依赖局部对话窗口，难以形成稳定、可演化的概念结构；其二，现有外部增强机制往往在“记忆存储”“知识图谱”“文档检索”“工具编排”之间摇摆，却缺少一层专门服务于理解的中间表示。本文提出 **World 0**，一种面向 LLM Agent 的概念世界认知层。World 0 不以事实存储为中心，也不以传统本体建模为目标，而是将 **概念（Concept）**、**关系（Relation）**、**上下文（Context）**、**激活（Activation）** 与 **投影（Projection）** 组织为一套可计算、可持久化、可由 LLM 读写的认知结构。其运行机制包括：从文本与对话中抽取结构化观察、在概念网络上执行关系强化与共现学习、在任务条件下进行扩散激活、生成局部概念投影视图，并通过衰减与成熟度演化维持结构稳定。本文阐述 World 0 的问题定义、系统模型、运行流程、实现机制与评测框架，并提出一套针对 “使用 World 0 / 不使用 World 0” 的 Agent 消融测试方案。我们认为，World 0 的核心价值不在于扩大存储规模，而在于为 Agent 提供一个任务条件下可投影、可解释、可演化的认知支架。

**关键词：** LLM Agent，Concept World，Cognitive Layer，Projection，Contextual Activation，Structured Understanding

## 1. 引言

随着大语言模型（LLM）逐渐从单轮问答系统演化为具有规划、工具使用与长期任务处理能力的 Agent，如何为 Agent 提供稳定、任务相关、可积累的认知结构，成为一个核心问题。当前主流方案大致分为四类：

1. 基于对话历史或摘要的上下文压缩方案；
2. 基于向量检索的相似内容召回方案；
3. 基于知识图谱或实体关系图的结构化方案；
4. 基于记忆库、日志库或文档库的外部存储方案。

这些方案各有价值，但也存在明显不足。纯历史摘要保留了语言连续性，却难以保持概念边界；向量检索能够召回相似片段，却难以保证结构解释性；传统知识图谱强调事实与实体，未必适合 Agent 的任务理解；记忆库和文档库则更接近存储系统，而非认知系统。

Agent 在真实任务中需要回答的，并不只是“我曾经看过什么”，而更常是：

- 当前任务中哪些概念正在发挥作用？
- 这些概念之间的关键关系是什么？
- 哪些结构边界决定了当前问题的理解方式？
- 在不暴露整个知识世界的情况下，应该向模型注入怎样的局部认知视图？

基于这一观察，本文提出 **World 0**：一个并不追求“覆盖整个世界”的系统，而是追求“在任务中产生可用局部世界”的认知层。World 0 的设计目标不是替代知识库、记忆系统或检索系统，而是在它们之上提供一层 **概念优先、关系优先、上下文敏感、投影导向** 的理解结构。

## 2. 问题定义

### 2.1 从存储问题到理解问题

本文将 Agent 面临的问题区分为以下几个层级：

- **存储问题**：信息如何被保存；
- **记忆问题**：过去事件如何被回忆；
- **事实问题**：实体与事实如何被表示；
- **认知问题**：任务当前应如何被理解。

World 0 只显式解决第四类问题。它不试图成为事实数据库，也不替代事件记忆，而是试图为 Agent 建立一个“面向任务理解的概念世界”。

### 2.2 形式化目标

给定一组输入观察 $O = \{o_1, o_2, ..., o_n\}$，每个观察可能包含自然语言文本、对话片段、结构化概念对和任务标签，系统需要构建一个概念世界 $W$，使得在任意任务上下文 $c$ 下，系统能够生成一个局部投影 $P_c(W)$，满足：

1. **相关性**：投影中的概念与当前任务高度相关；
2. **结构性**：投影不仅包含概念，还包含可解释的关系结构；
3. **局部性**：投影视图紧凑，不暴露整个全局结构；
4. **演化性**：世界结构可随新观察持续更新与重整；
5. **可操作性**：LLM 可直接读取投影结果并用于推理。

## 3. 核心概念

World 0 建立在五个一等概念之上。

### 3.1 Concept

Concept 是最小认知单元。一个概念不是简单词项，而是一个可被链接、激活、强化、衰减和投影的语义单位。概念可以表示对象、过程、方法、角色、原则或问题空间中的抽象结构。

### 3.2 Relation

Relation 是概念之间的 typed connection。World 0 不把关系视为附属元数据，而将其作为系统核心组成部分。关系可以表示依赖、包含、支持、激活、对比、先后、派生等语义结构。

### 3.3 Context

Context 不是一个静态标签，而是决定“当前哪些概念与关系更重要”的条件变量。上下文会影响激活传播、候选概念排序与投影结果。

### 3.4 Activation

Activation 是概念世界的运行机制。输入任务和种子概念触发局部区域的激活扩散，使系统能够从一个较大的概念网络中选出最有认知价值的局部区域。

### 3.5 Projection

Projection 是系统的操作输出。Agent 不直接读取完整世界，而是读取一份受任务条件约束的局部概念视图。World 0 的最终目标不是构造最大图，而是生成最有用的局部世界。

## 4. 设计原则

World 0 采用以下基本原则：

### 4.1 概念优先，而非事实优先

系统以概念为核心对象，而不是以事实三元组为中心。事实可以附着于概念或由关系支持，但它们不是核心抽象。

### 4.2 关系优先，而非分类优先

World 0 不否认层级结构，但不将 taxonomy 视为唯一结构。相似、对比、依赖、激活、支持等关系与包含关系同等重要。

### 4.3 上下文敏感，而非全局静态

同一概念在不同任务中的重要性应发生变化，系统不假定“概念重要性”具有全局固定排序。

### 4.4 为理解服务，而非为存储服务

World 0 的目标是提供理解支架，而非归档所有信息。因此它强调局部投影、激活区域与认知组织，而不是全量覆盖。

### 4.5 对 LLM 可操作

系统中的对象与输出必须既适于程序计算，又适于被语言模型读取与扩展。这意味着数据结构需要在形式性与语义可读性之间保持平衡。

## 5. 系统架构

World 0 在实现上可分为六个层级：

1. **输入层**：文本、对话、结构化观察；
2. **抽取层**：从原始输入中得到概念与关系；
3. **世界层**：管理概念节点与关系边；
4. **动力学层**：包含激活、共现学习、衰减与生命周期；
5. **投影层**：根据任务生成局部认知视图；
6. **接口层**：为 Python API、CLI、Web/UI 与 Agent 封装提供调用面。

当前实现中，对应代码模块主要包括：

- `world.py`：统一入口；
- `concepts/`：概念生命周期管理；
- `relations/`：关系生命周期管理；
- `dynamics/`：激活、Hebbian 学习、衰减、成熟度演化；
- `projection/`：投影生成；
- `extraction/`：LLM 驱动的概念提取；
- `store/`：JSON 持久化；
- `agents/`：面向 Agent 的高层交互层。

## 6. 运行机制

### 6.1 观察输入

World 0 的标准输入是 `Observation`，它包含：

- 概念列表；
- 关系列表；
- 可选概念描述；
- 任务标签；
- 来源标识；
- 时间戳。

当输入是自然语言文本或对话时，系统首先通过 LLM 将其转换为结构化 `Observation`。这一步不负责长期认知计算，只负责从语言中抽取概念候选与关系候选。

### 6.2 概念提取

概念提取的提示词要求模型返回严格 JSON，包含：

- `concepts`
- `relations`

系统在本地执行二次校验：

- 关系两端必须同时出现在概念列表中；
- 非法关系类型回退为 `related_to`；
- 非法 JSON 或噪声响应会被丢弃。

因此，提取链路的结构可表示为：

$$
\text{Dialogue/Text} \rightarrow \text{LLM Extraction} \rightarrow \text{Observation Validation}
$$

### 6.3 摄入（Ingest）

当一个 `Observation` 被摄入时，系统执行以下步骤：

1. 对每个概念执行 `get-or-create`；
2. 强化已有概念的激活计数与置信度；
3. 对显式关系执行创建或强化；
4. 对同轮共现概念执行 Hebbian 学习；
5. 更新概念描述；
6. 将脏对象批量刷盘。

这一机制意味着 World 0 既支持“用户明确声明的关系”，也支持“系统从重复共现中逐步发现的关系”。

### 6.4 Hebbian 共现学习

World 0 引入了受阈值约束的共现学习：如果两个概念在多次观察中共同出现，则它们之间会自动形成弱 `related_to` 关系。为了防止图结构爆炸，系统设置：

- 最小共现次数阈值；
- 单次学习中最大 pair 数限制；
- 自动关系较低的初始权重与较低的上限。

这种设计保留了结构成长能力，同时避免一次偶然共现生成大量噪音边。

### 6.5 激活传播

投影前，系统从种子概念出发进行扩散激活。传播强度由以下因子共同决定：

- 源节点当前激活值；
- 关系类型系数；
- 关系权重；
- 邻居概念 confidence；
- 深度衰减；
- 任务匹配加权；
- 概念 freshness；
- 关系 freshness。

可将其抽象表示为：

$$
a_{t+1}(v_j) = a_t(v_i) \cdot w_{ij} \cdot r_{type} \cdot c_j \cdot d_t \cdot \gamma_{task} \cdot \tau_{rel} \cdot \tau_{node}
$$

其中：

- $a_t(v_i)$ 为当前节点激活值；
- $w_{ij}$ 为边权；
- $r_{type}$ 为关系类型传播系数；
- $c_j$ 为邻居节点 readiness；
- $d_t$ 为深度衰减；
- $\gamma_{task}$ 为任务亲和度；
- $\tau_{rel}, \tau_{node}$ 分别为关系与节点的新鲜度权重。

这使得 World 0 的传播不只是“图上走几步”，而是任务感知、时间感知、关系感知的局部认知扩散。

### 6.6 投影生成

在得到候选激活集合后，系统不直接按分数截断，而使用 MMR（Maximal Marginal Relevance）进行选择，使投影同时满足：

- 高相关性；
- 低冗余；
- 任务条件偏置；
- 对近期活跃概念的适度偏好。

最终输出 `Projection` 对象，包括：

- 概念集合；
- 内部关系集合；
- 激活分数；
- 任务标签。

系统提供 `render()` 接口将投影渲染为 Markdown，以便直接注入 Agent prompt。

### 6.7 反思与演化

World 0 通过 `reflect()` 执行周期性结构维护，包括：

- 时间衰减；
- 成熟度晋升；
- 噪音修剪。

概念成熟度包含：

- `embryonic`
- `developing`
- `established`
- `core`
- `fading`

这使概念世界具有“生长-稳定-退化-再激活”的动态特征，而非一次性静态图结构。

## 7. World 0 在 LLM Agent 中的使用方式

World 0 在 LLM Agent 中并不是模型内部机制，而是位于模型外部的一层认知中间件。其调用分为两个方向：

### 7.1 写入方向：从语言到概念世界

$$
\text{Dialogue/Text} \rightarrow \text{LLM Extraction} \rightarrow \text{Observation} \rightarrow \text{World Ingest}
$$

在这一方向上，LLM 用于从自然语言中抽取结构化概念与关系，而 World 0 负责把这些结果并入可演化的认知结构。

### 7.2 读取方向：从概念世界到模型提示

$$
\text{User Query} \rightarrow \text{Seed Extraction} \rightarrow \text{World Projection} \rightarrow \text{Prompt Injection} \rightarrow \text{LLM Response}
$$

在这一方向上，World 0 先为当前任务生成局部概念视图，再将其作为认知上下文注入给模型。模型不是直接面对整个历史，而是面对一份结构化投影。

## 8. 与现有方案的区别

### 8.1 与记忆系统的区别

记忆系统回答“发生了什么”；World 0 回答“应如何理解”。前者强调事件与回忆，后者强调概念组织与任务投影。

### 8.2 与知识图谱的区别

知识图谱通常围绕实体与事实展开，而 World 0 围绕概念与认知相关性展开。它不以事实完备性为主要目标，而以局部世界的可用性为目标。

### 8.3 与向量检索的区别

向量检索擅长相似内容召回，但常缺乏显式结构解释。World 0 的核心输出不是相似片段，而是任务相关的概念关系视图。

### 8.4 与本体工程的区别

传统本体强调形式逻辑与严格范畴。World 0 借鉴其结构意识，但不追求高强度形式化推理，而追求 LLM 可操作性与任务认知价值。

## 9. 评测框架

World 0 的评测不应只关注速度，还应关注认知质量。一个合理的评测框架至少包括以下维度。

### 9.1 概念完整性

- 同名或大小写变体是否归并；
- 相邻但不同概念是否保持区分；
- 概念成熟度是否与使用频率相匹配。

### 9.2 关系质量

- 投影是否保留 typed relations；
- 自动关系是否被限制在合理范围内；
- 跨域桥接关系是否可解释。

### 9.3 上下文敏感性

- 相同 seed 在不同 task 下是否产生不同投影；
- 任务对概念排序与集合选择是否有真实影响。

### 9.4 投影质量

- 投影 precision / recall；
- 概念覆盖度；
- 结构连通性；
- 冗余控制。

### 9.5 演化稳定性

- 多轮 ingest 后结构是否稳定增长；
- `reflect()` 是否有效清理噪音；
- 衰减曲线是否与设计预期一致。

## 10. Agent 提升的对照实验设计

为了验证 World 0 对 Agent 的真实增益，本文建议采用消融实验而非单独功能测试。实验应至少包含三个条件组：

1. **No Memory**：不给历史结构；
2. **Raw History Baseline**：给原始历史或普通摘要；
3. **World0 Projection**：使用 World 0 进行沉淀与投影。

### 10.1 控制变量

所有实验组必须保持以下因素一致：

- 模型版本；
- 任务集；
- token 预算；
- 最大交互轮数；
- 系统提示词骨架；
- 工具可用性。

唯一变化应是“是否使用 World 0 及其投影”。

### 10.2 任务集类型

推荐采用以下任务类型：

- 多轮累积任务；
- 跨会话桥接任务；
- 长距离概念召回任务；
- 关系推理任务；
- 上下文切换任务。

### 10.3 评价指标

建议记录：

- 最终任务成功率；
- 概念 precision / recall；
- 关键关系命中率；
- 上下文聚焦度；
- token-per-success；
- latency-per-success。

### 10.4 预期假设

本文提出以下待验证假设：

- H1：World 0 能提高多轮任务中的结构化正确率；
- H2：World 0 能提高跨域桥接任务中的关系识别率；
- H3：World 0 能在相同 token 预算下生成更高聚焦度的上下文；
- H4：World 0 的增益主要来自“结构化投影”，而不仅仅是“提供了更多历史文本”。

## 11. 局限性

尽管 World 0 提供了一种新的认知层思路，但其当前形式仍存在局限。

### 11.1 对 LLM 抽取质量的依赖

概念与关系的初始质量受到抽取模型能力影响。虽然系统有本地校验，但仍无法完全消除语义抽取偏差。

### 11.2 关系语义仍相对有限

当前关系类型集合有限，尚未覆盖更复杂的因果、约束、角色化和层级语义。

### 11.3 当前上下文建模仍偏轻量

现有任务上下文主要通过 task history 和 task affinity 间接体现，尚未形成更显式、更可组合的上下文模型。

### 11.4 投影仍以局部启发式为主

当前投影依赖激活传播与 MMR 选择，是一种有效但相对轻量的策略。未来仍可探索更强的投影成形机制。

## 12. 未来工作

未来可沿以下方向扩展：

1. 显式上下文对象与上下文图；
2. perspective profile 与角色条件投影；
3. 更丰富的 relation typing 与 relation refinement；
4. 基于真实 Agent 任务的消融评测基准；
5. 与外部记忆系统、文档系统和检索系统的分层协作；
6. 更形式化的投影优化目标与可学习打分函数。

## 13. 结论

本文提出了 World 0，一种面向 LLM Agent 的概念世界认知层。与传统知识图谱、记忆系统和检索系统不同，World 0 并不以事实覆盖或存储规模为核心，而以 **概念结构、关系组织、任务激活与局部投影** 为核心。其关键贡献在于：

- 将 Agent 的“理解问题”与“存储问题”明确分离；
- 提供一套概念优先、关系优先、上下文敏感的中间表示；
- 建立从自然语言输入到结构化认知世界，再到任务投影输出的完整链路；
- 为评估 Agent 在使用结构化认知层前后的差异提供实验基础。

World 0 的根本目标不是构建一个最大、最全的世界模型，而是为 Agent 提供一个在任务当下“足够小、足够准、足够有结构”的局部世界。我们认为，这种从“存储增强”走向“认知增强”的转向，是 LLM Agent 基础设施演进中值得进一步系统研究的方向。
